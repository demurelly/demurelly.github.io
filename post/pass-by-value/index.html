Pass-By-Value

传递方式

Java中参数传递都是值传递，参见下面代码。

public class PassByRerOrVal {
    private String name;
    public PassByRerOrVal(String n){this.name = n;}

    public String getName() {
        return name;
    }
    public void setName(String name) {
        this.name = name;
    }
    public static void main(String args[]){
        PassByRerOrVal init = new PassByRerOrVal("init");
        change(init);
        System.out.println("changed : " + init.getName());  // changed : init
        modify(init);
        System.out.println("modify : " + init.getName());   // modify : modify ref
    }
    private static void change(PassByRerOrVal change){
        PassByRerOrVal t = new PassByRerOrVal("change ref");
        change = t;
    }
    private static void modify(PassByRerOrVal p){
        p.setName("modify ref");
    }

}

传递流程

  • 定义init为PassByRerOrVal类型并且属性name值为init ，change()参数change对应的值为null
  • 当change()被调用时，参数change将会依赖init。
  • change()中定义t变量并把值赋给change之后，change将不再指向init变量地址，而是指向t变量。此时对change做的所有改变都不会影响到init变量
  • 而modify则是直接修改内部属性。

final修饰参数

从上面可以知道Java中参数传递都是值传递，做的所有修改不会影响原变量，那为何需要final来修饰参数呢？还是说final修饰的参数，内部属性也不允许修改？

public static void main(String args[]){
    PassByRerOrVal p = new PassByRerOrVal("init");
    finalChange(p);
    System.out.println(p.getName()); // final...
}
private static void finalChange(final PassByRerOrVal fChange){
    fChange.setName("final...");
}

从上面代码可以知道，即使是final修饰的参数，内部属性仍然是可以被修改的。那么final修饰参数的意义又是什么呢？通过查阅资料，了解到final修饰参数是为了保证内部类中参数的一致性。

public static void main(String args[]){
    finalChange("hello");
}
private static void finalChange( String str){
    AClass ac = new AClass() {
        @Override
        void something() {
            str = "hehe";
        }
    };
}

上面这段代码在JDK8中已经不能编译通过，提示通过内部类访问str变量，该变量必须是final修饰的变量或者实际最终变量，既没有被修改过的。
